### Embedded AI systems

- **The fresh embedded-adjacent read is ENPIRE's resource accounting, and it is a warning for anyone selling "agents will run the robot."** The paper measures Mean Robot Utilization and Mean Token Utilization and finds that as you scale the robot fleet, robot utilization falls while token consumption climbs, because the agents spend more time reading logs and summarizing each other than operating hardware. Larger fleets reach success sooner but burn far more tokens. The honest takeaway is that physical autoresearch is bottlenecked by compute and coordination, not just robots, which is the same edge-budget question the WAM cluster keeps circling. [ENPIRE, NVIDIA GEAR](https://research.nvidia.com/labs/gear/enpire/)

### Research papers worth a look

- **ENPIRE: Agentic Robot Policy Self-Improvement in the Real World (NVIDIA, CMU, UC Berkeley).** A harness framework with four modules (Environment for auto reset and verification, Policy Improvement, Rollout on single or parallel robots, Evolution where agents analyze logs and consult literature) that lets frontier coding agents run real-world robot research autonomously and reach 99% pass@8 on dexterous tasks. It proposes Mean Robot Utilization and Mean Token Utilization as efficiency metrics for multi-agent physical autoresearch. Two things make it worth a careful read for a data business. The automated verifier is outcome-QC without a human, and the limitations section quantifies that scaling robots drives token cost up while robot utilization falls. Read it as the autoresearch loop crossing into hardware, where the scarce inputs become a trustworthy reset-and-verify interface and clean data an agent can act on. [ENPIRE, NVIDIA GEAR](https://research.nvidia.com/labs/gear/enpire/)
